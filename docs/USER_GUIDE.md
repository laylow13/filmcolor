# Filmcolor 使用手册

Filmcolor 是一个本地胶片去色罩工作台。它不会修改你的原始照片，所有调校决定记录在 JSON 附属文件中，每次渲染都从原始文件和参数重新生成，结果可复现。

## 安装与启动

### 环境准备

需要 Python 3.13 和 Node.js 22+。

```powershell
# 首次安装
git clone https://github.com/laylow13/filmcolor.git
cd filmcolor

# Python 依赖
uv sync --extra dev

# 前端依赖
cd web
npm install
cd ..
```

### 启动

打开两个终端：

```powershell
# 终端 1：后端 API
uv run uvicorn filmcolor_server.app:app --reload --host 127.0.0.1 --port 8000
```

```powershell
# 终端 2：前端界面
cd web
npm run dev
```

浏览器打开 **http://127.0.0.1:5173**。

### （可选）启用 NegPy 实验引擎

NegPy 是一个更专业的第三方色彩负片处理引擎，可作为实验性预览后端。

```powershell
git submodule update --init --recursive
uv sync --extra dev --extra negpy
```

不安装 NegPy 不影响正常使用，Filmcolor 自带引擎随时可用。

## 界面概览

界面分为三栏：

| 区域 | 功能 |
|------|------|
| 左侧边栏 | 已导入的胶卷列表 |
| 中央 | 预览区 + 底片夹（帧缩略图） |
| 右侧面板 | 引擎选择、采样工具、输出风格、参数读数 |

## 基本工作流程

### 1. 导入胶卷

将原始扫描文件放在一个文件夹中，然后通过 API 导入：

```
POST /api/rolls/import
{"source_dir": "D:/扫描件/2026-05-10-第一卷", "name": "第一卷"}
```

支持格式：PNG、JPEG、TIFF、CR2、CR3、NEF、ARW、DNG、ORF、RAF、RW2。

导入后左侧出现胶卷名称，点击进入。

### 2. 浏览底片

中央底部的 Contact Sheet 显示该卷所有帧。每帧标注状态：

- **unprocessed** — 未处理
- **auto_processed** — 已自动处理
- **manually_adjusted** — 已手动调校
- **failed** — 处理失败

点击任意帧选中，预览区显示渲染结果。

### 3. 渲染预览

选中帧后，点击工具栏的 ▶ 按钮渲染预览。首次渲染使用自动估计的去色罩参数。

### 4. 选择引擎

右侧面板顶部 **ENGINE** 区域：

- **Filmcolor**（默认）— 自带的轻量去色罩引擎
- **NegPy Experimental**（需安装）— 第三方专业引擎，CPU 处理

切换引擎后需重新渲染。

### 5. 放置采样点

这是校正色彩的核心操作。

在 **SAMPLES** 区域选择采样类型：

| 类型 | 用途 | 标记色 |
|------|------|--------|
| **Film Base** | 标记底片的橙色色罩区域（片基），引擎据此计算色罩补偿 | 琥珀色 |
| **Gray** | 标记场景中本应是中性灰的区域，引擎据此做白平衡 | 灰色 |
| **White** | 标记场景中最亮的参考白点，引擎据此调整白点 | 白色 |

操作步骤：
1. 先渲染预览（点击 ▶）
2. 在 SAMPLES 区选择采样类型
3. 在预览图上点击放置采样点
4. 彩色圆点标记出现在点击位置
5. 预览会在 500ms 后自动重新渲染，应用新的采样参数

采样列表显示所有已放置的采样点，点击 ✕ 可删除单个采样。

#### 采样建议

- **Film Base**：点击胶片边框或未曝光的片基区域。2-3 个点通常足够。放在不同亮度区域效果更好。
- **Gray**：点击画面中你知道本应是中性灰色的物体（如水泥路面、阴影中的白墙）。1-2 个点即可。
- **White**：点击画面中最亮的白色或近白色区域。1 个点即可。

### 6. 切换输出风格

**STYLE** 区域提供三种风格：

| 风格 | 特点 |
|------|------|
| **faithful** | 忠实还原胶片特性，自然过渡 |
| **neutral** | 低对比、中性色调，适合后期再编辑 |
| **share** | 增强对比和饱和度，适合直接分享 |

### 7. 同步设置到其他帧

一卷胶片通常拍摄条件相同。你只需要精细调校一两帧，然后把设置同步到其余帧：

1. 选一帧，调好采样点和风格
2. **Ctrl + 点击**（Mac: Cmd + 点击）要同步的目标帧
3. 选中的帧出现橙色边框
4. 顶部出现 "Sync from <帧号>" 按钮，显示已选帧数
5. 点击按钮，源帧的采样点和风格设置复制到所有选中帧

同步内容包括：采样点坐标、输出风格、曝光补偿、对比度。

## 理解面板读数

Filmcolor 引擎模式下，右侧面板底部显示当前帧的参数：

| 读数 | 含义 |
|------|------|
| **Mask confidence** | 色罩估计的置信度。1.0 = 完全来自用户采样，0.55 = 自动灰度世界估计 |
| **Exposure** | 曝光补偿值（EV） |
| **Contrast** | 对比度调整值 |

NegPy 引擎模式下，显示 NegPy 的 Backend、Commit、Adapter 信息。

## 故障排除

### 前端显示 "No rolls imported"

尚未导入任何胶卷。使用 API 导入一个文件夹。

### 预览渲染失败

- 确认原始文件未移动或删除
- 检查文件格式是否在支持列表中
- RAW 文件可能需要安装 `rawpy`：`uv sync --extra dev --extra raw`

### NegPy 显示为不可用

NegPy 引擎显示灰色并带有原因说明，常见原因：
- 未初始化子模块：运行 `git submodule update --init --recursive`
- 未安装依赖：运行 `uv sync --extra dev --extra negpy`

### 采样点位置不准确

采样坐标以原始图像尺寸为基准。如果你更换了不同分辨率的预览（如切换 max_size），坐标会自动换算。
